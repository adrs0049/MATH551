Forward and Backward Error

University of Massachusetts Amherst
Big Idea

Forward error asks: “How wrong is my answer?” Backward error asks: “What problem did I actually solve?” A stable algorithm solves a nearby problem exactly—and the condition number determines whether “nearby problem” means “nearby answer.”

A Motivating Example¶

Consider computing f(x)=x+1−xf(x) = \sqrt{x+1} - \sqrt{x}f(x)=x+1​−x​ for large xxx.

The condition number is κ≤1/2\kappa \leq 1/2κ≤1/2 — this is a well-conditioned problem!

Yet using 4-digit decimal arithmetic:

f(1984)=1985−1984=44.55−44.54=0.01f(1984) = \sqrt{1985} - \sqrt{1984} = 44.55 - 44.54 = 0.01f(1984)=1985​−1984​=44.55−44.54=0.01
(1)

The true value is 0.01122. We lost two significant digits — 10% relative error from a well-conditioned problem!

What went wrong? The algorithm subtracts two nearly-equal numbers, amplifying their individual roundoff errors. This is called subtractive cancellation.

A stable alternative: Rationalize the expression:

x+1−x=(x+1−x)(x+1+x)x+1+x=1x+1+x=189.09=0.01122✓\sqrt{x+1} - \sqrt{x} = \frac{(\sqrt{x+1} - \sqrt{x})(\sqrt{x+1} + \sqrt{x})}{\sqrt{x+1} + \sqrt{x}} = \frac{1}{\sqrt{x+1} + \sqrt{x}} = \frac{1}{89.09} = 0.01122 \quad \checkmarkx+1​−x​=x+1​+x​(x+1​−x​)(x+1​+x​)​=x+1​+x​1​=89.091​=0.01122✓
(2)

Same problem, different algorithm, full accuracy.

This example reveals something fundamental: not all errors are the problem’s fault. We need a framework to separate algorithm quality from problem sensitivity.

Forward Error¶

Consider computing y=f(x)y = f(x)y=f(x). Due to roundoff or approximation, we actually compute y~\tilde{y}y~​.

Definition 1 (Forward Error)
Forward error=∣y~−y∣=∣y~−f(x)∣\text{Forward error} = |\tilde{y} - y| = |\tilde{y} - f(x)|Forward error=∣y~​−y∣=∣y~​−f(x)∣
(3)

Question answered: How far is our answer from the truth?

Forward error is the most intuitive measure—it directly answers “how wrong is my answer?”

Absolute vs. Relative Error¶

For meaningful comparisons, we often use relative error:

Definition 2 (Relative Forward Error)
Relative forward error=∣y~−y∣∣y∣\text{Relative forward error} = \frac{|\tilde{y} - y|}{|y|}Relative forward error=∣y∣∣y~​−y∣​
(4)

This measures error as a fraction of the true value.

Example: An error of 0.001 means different things for:

  • y=1y = 1y=1: relative error is 0.1% (excellent)

  • y=0.001y = 0.001y=0.001: relative error is 100% (useless)

Relative error captures this distinction. When we say “accurate to 6 digits,” we mean relative error ≈10−6\approx 10^{-6}≈10−6.

Definition 3 (Forward Stable Algorithm)

An algorithm is forward stable if:

∣y~−y∣∣y∣=O(εmach⋅κ)\frac{|\tilde{y} - y|}{|y|} = O(\varepsilon_{\text{mach}} \cdot \kappa)∣y∣∣y~​−y∣​=O(εmach​⋅κ)
(5)

The forward error is bounded by machine precision times the condition number.

The Catch¶

Forward error seems like the natural measure—we want to know how wrong we are! But there’s a problem:

Forward error requires knowing the true answer y=f(x)y = f(x)y=f(x).

If we knew the true answer, we wouldn’t need to compute it. This is where backward error comes in.

Backward Error¶

Instead of asking “how wrong is my answer?”, backward error asks a different question.

Definition 4 (Backward Error)
Backward error=min⁡{∣x~−x∣:f(x~)=y~}\text{Backward error} = \min\{|\tilde{x} - x| : f(\tilde{x}) = \tilde{y}\}Backward error=min{∣x~−x∣:f(x~)=y~​}
(6)

Question answered: What input would give our output exactly?

In other words: our computed y~\tilde{y}y~​ might not equal f(x)f(x)f(x), but it does equal f(x~)f(\tilde{x})f(x~) for some nearby x~\tilde{x}x~. The backward error measures how far x~\tilde{x}x~ is from xxx.

Why Backward Error?¶

At first, backward error seems less useful than forward error. But it turns out to be more fundamental for algorithm analysis. Here’s why:

1. Backward error is computable

Forward error requires knowing the true answer—but if we knew that, we wouldn’t need to compute it!

Backward error, on the other hand, is often directly computable:

  • Linear systems: Given computed x~\tilde{x}x~, the residual r=b−Ax~r = b - A\tilde{x}r=b−Ax~ tells us backward error immediately—we solved Ax~=b−rA\tilde{x} = b - rAx~=b−r exactly.

  • Root finding: Given approximate root r~\tilde{r}r~, the residual f(r~)f(\tilde{r})f(r~) measures backward error.

2. Backward error separates algorithm from problem

There are two sources of error:

  • Algorithm quality: Does the algorithm solve some nearby problem accurately?

  • Problem sensitivity: How much does the answer change when the problem changes slightly?

Backward error isolates the first question. If an algorithm has small backward error, it’s doing its job well—any remaining forward error is the problem’s fault (ill-conditioning), not the algorithm’s.

3. Backward error accounts for input uncertainty

Real-world inputs have measurement error. If your input xxx has uncertainty δ\deltaδ, then:

  • You don’t actually know you’re solving f(x)f(x)f(x)—you might be solving f(x+ϵ)f(x + \epsilon)f(x+ϵ) for some ∣ϵ∣≤δ|\epsilon| \leq \delta∣ϵ∣≤δ

  • An algorithm with backward error ≤δ\leq \delta≤δ is as good as exact for your purposes

Principle: Don’t ask for more accuracy than your inputs justify.

Key insight: Backward stability implies forward stability (for well-conditioned problems). This is why we focus on backward stability—it’s the stronger, more useful guarantee.

The Relationship: Forward Error ≤ κ × Backward Error¶

Now we connect backward error to forward error using the condition number from the previous section.

Derivation¶

Suppose we compute y~\tilde{y}y~​ instead of the true y=f(x)y = f(x)y=f(x). If the algorithm is backward stable:

y~=f(x~)for some x~ near x\tilde{y} = f(\tilde{x}) \quad \text{for some } \tilde{x} \text{ near } xy~​=f(x~)for some x~ near x
(8)

The forward error is the difference ∣y~−y∣=∣f(x~)−f(x)∣|\tilde{y} - y| = |f(\tilde{x}) - f(x)|∣y~​−y∣=∣f(x~)−f(x)∣.

By Taylor expansion:

∣f(x~)−f(x)∣≈∣f′(x)∣⋅∣x~−x∣|f(\tilde{x}) - f(x)| \approx |f'(x)| \cdot |\tilde{x} - x|∣f(x~)−f(x)∣≈∣f′(x)∣⋅∣x~−x∣
(9)

Converting to relative errors:

∣y~−y∣∣y∣≈∣f′(x)∣∣f(x)∣⋅∣x~−x∣=∣xf′(x)f(x)∣⏟κ⋅∣x~−x∣∣x∣\frac{|\tilde{y} - y|}{|y|} \approx \frac{|f'(x)|}{|f(x)|} \cdot |\tilde{x} - x| = \underbrace{\left|\frac{x f'(x)}{f(x)}\right|}_{\kappa} \cdot \frac{|\tilde{x} - x|}{|x|}∣y∣∣y~​−y∣​≈∣f(x)∣∣f′(x)∣​⋅∣x~−x∣=κ∣∣​f(x)xf′(x)​∣∣​​​⋅∣x∣∣x~−x∣​
(10)
Theorem 1 (The Golden Rule)
∣y~−y∣∣y∣⏟relative forward error≲κ⏟condition number⋅∣x~−x∣∣x∣⏟relative backward error\underbrace{\frac{|\tilde{y} - y|}{|y|}}_{\text{relative forward error}} \lesssim \underbrace{\kappa}_{\text{condition number}} \cdot \underbrace{\frac{|\tilde{x} - x|}{|x|}}_{\text{relative backward error}}relative forward error∣y∣∣y~​−y∣​​​≲condition numberκ​​⋅relative backward error∣x∣∣x~−x∣​​​
(11)

Forward error ≤ Condition number × Backward error

This is the central equation of numerical analysis. It cleanly separates:

  • Problem sensitivity (κ) — intrinsic to the mathematics

  • Algorithm quality (backward error) — what we can control

What This Means¶

ComponentControlled byCan we improve it?
Forward errorBothIndirectly
Condition number κ\kappaκThe problemNo (it’s math)
Backward errorThe algorithmYes!

Practical implications:

  • Well-conditioned + backward stable → accurate: Small κ\kappaκ and small backward error guarantee small forward error

  • Ill-conditioned → trouble: Even perfect algorithms give poor forward error when κ\kappaκ is large

  • Focus on backward stability: Write algorithms with small backward error; that’s all you can control

Unstable Algorithms: Common Pitfalls¶

Even well-conditioned problems can produce terrible results with unstable algorithms. Three operations are particularly dangerous:

OperationProblemStable Alternative
Subtracting close numbersRelative error explodesReformulate algebraically
Dividing by small numbersAmplifies numerator errorUse logarithms or rescale
Adding numbers of vastly different magnitudeSmall values lostSort and sum smallest first
Example: The Quadratic Formula

Solve x2−106x+1=0x^2 - 10^6 x + 1 = 0x2−106x+1=0. The roots are x1≈106x_1 \approx 10^6x1​≈106 and x2≈10−6x_2 \approx 10^{-6}x2​≈10−6.

Standard formula for the small root:

x2=106−1012−42≈106−1062=0x_2 = \frac{10^6 - \sqrt{10^{12} - 4}}{2} \approx \frac{10^6 - 10^6}{2} = 0x2​=2106−1012−4​​≈2106−106​=0
(12)

Catastrophic cancellation! The true value is 10-6, so we have 100% relative error.

Stable alternative: Use x1x2=c/a=1x_1 x_2 = c/a = 1x1​x2​=c/a=1, so x2=1/x1x_2 = 1/x_1x2​=1/x1​. Compute x1x_1x1​ (no cancellation), then x2=1/x1x_2 = 1/x_1x2​=1/x1​ with full precision.

Example: Computing ex−1e^x - 1ex−1

For small xxx, direct computation fails:

x = 1e-15
result = np.exp(x) - 1  # Returns ~1.1e-15 (only 1 correct digit!)

Why? e10−15≈1.000000000000001e^{10^{-15}} \approx 1.000000000000001e10−15≈1.000000000000001, stored as 1.0 due to limited precision. Then 1.0−1.0≈01.0 - 1.0 \approx 01.0−1.0≈0.

Stable alternative: Use the dedicated function:

result = np.expm1(1e-15)  # Returns 1.0e-15 (full precision!)
Example: Kahan Summation

When summing many numbers, roundoff errors accumulate. Kahan summation tracks and compensates for these errors:

def kahan_sum(x):
    """Compensated summation — much more accurate than naive sum."""
    s = 0.0  # Running sum
    c = 0.0  # Compensation for lost low-order bits
    for xi in x:
        y = xi - c        # Compensated value to add
        t = s + y         # New sum (may lose precision)
        c = (t - s) - y   # Recover what was lost
        s = t
    return s

This achieves O(εmach)O(\varepsilon_{\text{mach}})O(εmach​) error instead of O(n⋅εmach)O(n \cdot \varepsilon_{\text{mach}})O(n⋅εmach​) for naive summation.

Summary¶

ConceptMeasuresControlled byComputable?

Backward errorPerturbation to inputAlgorithmOften yes (residual)

The workflow:

  1. Check if the problem is well-conditioned (small κ\kappaκ)

  2. Use a backward-stable algorithm (small backward error)

  3. Then forward error will be small automatically

If forward error is large despite backward stability, the problem is ill-conditioned—reformulate if possible.
