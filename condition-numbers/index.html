Condition Numbers

University of Massachusetts Amherst
Big Idea

The condition number measures how sensitive a problem is to small perturbations in the input. Large condition numbers signal potential trouble regardless of the algorithm used—it’s a property of the mathematics, not the implementation.

Why Condition Numbers Matter¶

Before we discuss algorithms and their errors, we need to understand: some problems are inherently harder than others.

Consider computing f(x)=tan⁡(x)f(x) = \tan(x)f(x)=tan(x) near x=π/2x = \pi/2x=π/2. Even with infinite precision arithmetic and a perfect algorithm, tiny uncertainties in xxx cause huge changes in tan⁡(x)\tan(x)tan(x). This isn’t a flaw in our algorithm—it’s the nature of the tangent function near its pole.

The condition number quantifies this sensitivity. It answers: if my input is slightly wrong, how wrong might my output be?

Absolute and Relative Error¶

Before measuring problem sensitivity, we need precise definitions of error.

Definition 1 (Absolute and Relative Error)

Let xxx be the true value and x∗x^*x∗ be a numerical approximation.

Absolute error:

Abs(x)=∣x−x∗∣\text{Abs}(x) = |x - x^*|Abs(x)=∣x−x∗∣
(1)

Relative error:

Rel(x)=∣x−x∗∣∣x∣,x≠0\text{Rel}(x) = \frac{|x - x^*|}{|x|}, \quad x \neq 0Rel(x)=∣x∣∣x−x∗∣​,x=0
(2)

Why Relative Error Matters¶

x=1x = 1x=1, x∗=2x^* = 2x∗=2x=106x = 10^6x=106, x∗=106+1x^* = 10^6 + 1x∗=106+1

Both have the same absolute error, but the first is off by 100% while the second is nearly perfect. Relative error captures what matters.

Condition of fff at xxx¶

When we want to evaluate f(x)f(x)f(x), even if x∗x^*x∗ is close to xxx, the value f(x∗)f(x^*)f(x∗) may be far from f(x)f(x)f(x). The condition number quantifies this sensitivity.

Definition 2 (Condition Number)

Given a function f(x)f(x)f(x) and a fixed location xxx, the condition number is:

κ:=sup⁡∣x−x∗∣≠0∣Rel(f(x))Rel(x)∣=sup⁡∣x−x∗∣≠0∣f(x)−f(x∗)f(x)x−x∗x∣\kappa := \sup_{|x - x^*| \neq 0} \left| \frac{\text{Rel}(f(x))}{\text{Rel}(x)} \right| = \sup_{|x - x^*| \neq 0} \left| \frac{\frac{f(x) - f(x^*)}{f(x)}}{\frac{x - x^*}{x}} \right|κ:=∣x−x∗∣=0sup​∣∣​Rel(x)Rel(f(x))​∣∣​=∣x−x∗∣=0sup​∣∣​xx−x∗​f(x)f(x)−f(x∗)​​∣∣​
(3)

  • If κ≫1\kappa \gg 1κ≫1 (large), then evaluating fff at xxx is ill-conditioned

  • If κ≈O(1)\kappa \approx \mathcal{O}(1)κ≈O(1), then evaluating fff at xxx is well-conditioned

Simplified Formula via Taylor’s Theorem¶

Proposition 1 (Condition Number Formula)

For a differentiable function fff, the condition number at xxx is:

κ=∣xf′(x)f(x)∣\kappa = \left| \frac{x f'(x)}{f(x)} \right|κ=∣∣​f(x)xf′(x)​∣∣​
(4)
Proof 1

Since x∗x^*x∗ is close to xxx, we can write x∗=x+hx^* = x + hx∗=x+h and apply Taylor’s theorem:

f(x∗)=f(x+h)≈f(x)+f′(x)(x−x∗)f(x^*) = f(x + h) \approx f(x) + f'(x)(x - x^*)f(x∗)=f(x+h)≈f(x)+f′(x)(x−x∗)
(5)

This gives us:

f(x)−f(x∗)≈f′(x)(x−x∗)f(x) - f(x^*) \approx f'(x)(x - x^*)f(x)−f(x∗)≈f′(x)(x−x∗)
(6)

Substituting into the condition number definition:

κ=sup⁡x∣xf′(x)f(x)∣\kappa = \sup_x \left| \frac{x f'(x)}{f(x)} \right|κ=xsup​∣∣​f(x)xf′(x)​∣∣​
(7)

Examples¶

Example 2 (Square Root (Well-Conditioned))

Consider f(x)=xf(x) = \sqrt{x}f(x)=x​ with f′(x)=12xf'(x) = \frac{1}{2\sqrt{x}}f′(x)=2x​1​.

Near xˉ=1\bar{x} = 1xˉ=1, we have yˉ=f(xˉ)=1\bar{y} = f(\bar{x}) = 1yˉ​=f(xˉ)=1. Using a Taylor approximation:

y−yˉ=x−1≈12(x−1)=12(x−xˉ)y - \bar{y} = \sqrt{x} - 1 \approx \frac{1}{2}(x - 1) = \frac{1}{2}(x - \bar{x})y−yˉ​=x​−1≈21​(x−1)=21​(x−xˉ)
(8)

Variations in yyy are always smaller than variations in xxx. Computing the condition number:

κ=∣xf′(x)f(x)∣=∣x⋅12xx∣=12\kappa = \left| \frac{x f'(x)}{f(x)} \right| = \left| \frac{x \cdot \frac{1}{2\sqrt{x}}}{\sqrt{x}} \right| = \frac{1}{2}κ=∣∣​f(x)xf′(x)​∣∣​=∣∣​x​x⋅2x​1​​∣∣​=21​
(9)

Result: κ=1/2\kappa = 1/2κ=1/2 — evaluating x\sqrt{x}x​ is well-conditioned.

Example 4 (Logarithm Near 1 (Ill-Conditioned))

Consider f(x)=ln⁡(x)f(x) = \ln(x)f(x)=ln(x) near x=1x = 1x=1.

The condition number is a property of the mathematical problem—it tells us the best possible accuracy any algorithm could achieve. But even well-conditioned problems can give bad results if we use a bad algorithm. That’s the subject of the next section.
